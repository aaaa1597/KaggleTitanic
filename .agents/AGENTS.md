# Kaggle Titanic プロジェクトルール

このドキュメントは、Kaggle Titanic プロジェクトにおけるワークスペースのルール、環境の制約、およびガイドラインを規定したものです。

## 環境とパスの制約

> [!CAUTION]
> **仮想環境やプロジェクトを含むディレクトリパスに、日本語（非ASCII）文字を絶対に使用しないでください。**
> パスに非ASCII文字が含まれていると、C拡張ライブラリ（ScipyやScikit-LearnなどのDLL）のロードに失敗します。
> - **NG**: `D:\BizOwn\000_Biw2\38.データサイエンティストへの道\005_kaggle001\`
> - **OK**: `D:\BizOwn\000_Biw2\51_googleantigravity\1st_\`

- **Python バージョン**: Python 3.14.6
- **仮想環境**: `.venv`（プロジェクトルートに配置）
- スクリプトやコマンドを実行する際は、必ず `.venv` 内の Python インタープリタ（`.venv\Scripts\python.exe` または仮想環境のアクティベート）を使用してください。

## ディレクトリ構造

以下のディレクトリ構造を厳格に遵守してください：
- `data/raw/`: オリジナルの Kaggle データファイル（`train.csv`, `test.csv`, `gender_submission.csv`）。
- `data/processed/`: 抽出した特徴量、前処理済みのデータセット、生成した図やグラフ。
- `notebooks/`: 探索的データ分析（EDA）およびモデリング用の Jupyter Notebook（例：`titanic_eda.ipynb`）。
- `submissions/`: Kaggle 提出用の出力 CSV ファイル（例：`submission.csv`）。

## コーディングガイドライン

- Notebook 内はクリーンに保ってください。冗長なスクリプトセルを書くのではなく、前処理、特徴量エンジニアリング、評価のための関数を定義して使用してください。
- 新しいパッケージを導入した際は、必ず以下のコマンドで `requirements.txt` を更新してください：
  ```powershell
  .venv\Scripts\python.exe -m pip freeze > requirements.txt
  ```
- コミットメッセージや Kaggle への提出（submission）時のメッセージには、具体的な内容を記述してください。

## ユーザーからの要望
- 「手順nを実行して」って指示されたら、`Steps.md`ファイルの該当Noのセクションを実行すること。

# Zenn記事作成ルール
今後Zenn用の記事を作成・修正する際は、以下のルールを厳守すること。

## 1. 構成ルール
*   **冒頭の要約**: 記事の最初に必ず `# Abstract` セクションを設け、内容の要点を箇条書きで記載すること。

## 2. 文体・トーン
*   **絵文字の禁止**: 記事内では絵文字を使用しないこと。
*   **九州弁（博多弁・佐賀弁風）の使用**: 語尾に九州弁（例：「〜ばい」「〜やね」「〜と。」「〜ばってん」など）を自然に取り入れて記述すること。

## 3. 表記・フォーマット
*   **全角括弧の禁止**: 全角のカッコ「（）」は使用禁止とする。必ず半角のカッコ「()」を使用すること。
*   **リンク記法**: ファイルへのリンクは [ファイル名](file:///パス) 形式で記述すること。
